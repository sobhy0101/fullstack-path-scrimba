# Copilot Instructions - Learning Journal Solo Project

## Project Context

This is a **Responsive Design Solo Project** for the Scrimba Fullstack Path course. The student (Mahmoud 'Mike' Sobhy) is building a learning journal blog website with three responsive pages.

## Critical Project Details

### Design References
- **Figma File**: [Learning Journal Designs](https://www.figma.com/design/yqvEW2TqqqKXfUF5boidu3/Learning-Journal-Blog--Copy-?node-id=0-1&m=dev&t=Fzb27clv1piiCEz9-1)
- **Local Design Files**: Available in `design/svg/` and `design/png/` folders (6 files: 3 pages × 2 views)
- **Frame-Specific Links**: Available in README.md for each page (desktop + mobile)

### Technical Stack & Setup
- **Build Tool**: Vite (configured at parent directory: `fullstack-path-scrimba-git-clone`)
- **HTML/CSS**: Vanilla HTML5 + CSS3
- **Typography**: Google Fonts (Roboto, Merriweather)
- **Template System**: Use dynamic template loader (template-loader.js) for header/footer
- **Mobile-First**: Responsive design with media queries
- **Layout**: CSS Grid + Flexbox

### Pages Structure
1. **Homepage** (index.html) - Hero post + recent posts grid + "View More" button
2. **Post/Article Page** (article.html) - Full article + recent posts
3. **About Me Page** (about.html) - Profile + bio + recent posts

### Content Strategy
- **Blog Posts**: Create 9 posts total using responsive design learning concepts
- **Profile Image**: Use `images/my-profile-photo.webp`
- **Article Images**: Placeholder images (to be replaced later per stretch goals)
- **Icons**: Logo (logo.svg), Favicon (favicon.svg), Hamburger menu (create SVG)

### Key Requirements from README
- Mobile-first responsive design
- Relative units (rem, em, %, vh, vw)
- Semantic HTML5
- BEM methodology for CSS naming
- CSS Grid for layout
- Flexbox for posts/articles
- Viewport meta tag
- Open Graph + Twitter meta tags
- Alt attributes on images
- CSS reset/normalize
- External CSS file: `css/style.css`
- Comments explaining structure and styling

### Stretch Goals (for later phases)
- Custom content about student's learning journey
- "View More" JavaScript functionality (hide/show posts)
- Hamburger menu for mobile navigation
- Animations/transitions
- CSS variables
- Additional pages (contact, portfolio links)

## Code Style Guidelines

**CRITICAL**: Code must look human-written, not AI-generated

### Writing Style
- Use varied comment styles (some detailed, some brief, occasional typos)
- Natural variable/class naming (not overly systematic)
- Mix of coding patterns (not perfectly consistent)
- Occasional learning-oriented comments ("trying grid here", "learned this from Kevin Powell")
- Some CSS properties in different orders
- Mix of single and double quotes occasionally
- Real-world naming (not generic placeholder1, placeholder2)

- Learning responsive design from Scrimba course
- Instructors: Kevin Powell (CSS), Per Borgen, others

- Will share with Discord learning community
